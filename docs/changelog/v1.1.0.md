# v1.1.0 更新日志

发布日期：2026-03-04

---

## 新增功能

### 1. 支持单独执行 Maven 构建或 Docker 部署

新增 `--skip-maven` 和 `--skip-docker` 参数，允许跳过流水线中的某一阶段。

```bash
# 仅执行 Maven 构建，跳过 Docker 部署
local-deploy deploy --skip-docker
local-deploy --skip-docker

# 仅执行 Docker 构建和部署，跳过 Maven 构建
local-deploy deploy --skip-maven
local-deploy --skip-maven

# 组合使用：指定 profile 并跳过 Docker
local-deploy --profile prod --skip-docker
```

**使用场景：**
- 目标环境无 Docker 访问权限，仅需 Maven 打包
- Maven 构建已成功，因 Docker 配置错误需单独重新部署

---

### 2. 容器管理命令 (`container`)

新增 `container` 命令，支持对已部署容器的常用操作，无需手动拼接 Docker 命令。

容器名称自动从配置中推导：`{artifactId}-{profile}`。

```bash
local-deploy container stop       # 停止容器
local-deploy container start      # 启动容器
local-deploy container restart    # 重启容器
local-deploy container rm         # 删除容器
local-deploy container logs       # 查看日志
local-deploy container logs -f    # 实时追踪日志
local-deploy container status     # 查看容器状态
```

---

### 3. 自动构建开关 (`auto_deploy`)

新增 `auto_deploy` 配置项，控制 `git push` 时是否自动触发部署。

```yaml
# local-deploy.yml
auto_deploy: true    # 设为 false 则 git push 不会触发部署
```

- 设为 `false` 后，`git push` 时 pre-push hook 仍会执行，但会跳过部署流程并输出提示
- 手动执行 `local-deploy deploy` 不受影响
- 无需删除或修改 Git Hook 即可临时禁用自动部署

---

## 改进

- `--profile`、`--skip-maven`、`--skip-docker` 等部署参数支持作为顶层参数直接使用，无需显式写 `deploy` 子命令
- 更新 pre-push hook 模板，传递 `--hook` 标记以区分手动与自动触发
