# v1.1.1 更新日志

发布日期：2026-03-04

---

## 新增功能

### 1. 后台部署与日志记录

git push 触发的部署流程现在在后台执行，不再阻塞 push 操作。所有部署输出自动记录到日志文件。

**日志文件：** 项目根目录下的 `local-deploy.log`（与 `local-deploy.yml` 同级）

**特性：**
- 部署在后台运行（`nohup ... &`），git push 立即返回
- 部署输出（stdout + stderr）重定向到 `local-deploy.log`
- 每次执行覆盖之前的日志（覆盖策略）
- 适配第三方 Git 工具（IntelliJ IDEA、SourceTree 等）无法查看 hook 输出的场景

---

### 2. 部署日志查看命令 (`log`)

新增 `log` 命令，查看最近一次部署的日志输出。

```bash
# 查看最近一次部署日志
local-deploy log

# 实时跟踪部署日志（部署进行中时使用）
local-deploy log --follow
local-deploy log -f
```

**使用场景：**
- git push 后查看后台部署结果
- 使用 `--follow` 实时跟踪正在进行的部署

---

### 3. Hook 自动写入 Local-Deploy 安装路径

pre-push hook 模板新增 `LOCAL_DEPLOY_HOME` 变量，`init` 时自动填入 Local-Deploy 的实际安装路径。

hook 查找 `local-deploy.sh` 的优先级：
1. 全局命令（`command -v local-deploy`）
2. 项目根目录（`$PROJECT_ROOT/local-deploy.sh`）
3. 安装路径（`$LOCAL_DEPLOY_HOME/local-deploy.sh`）

**解决的问题：** 未全局安装时，hook 无法找到 `local-deploy.sh` 导致部署不执行。

---

### 4. `init` 命令支持 `--force` 强制覆盖

新增 `--force`（`-f`）参数，强制覆盖已有的配置文件和 pre-push hook。

```bash
# 强制重新初始化（覆盖已有的 hook 和配置文件）
local-deploy init --force
```

**解决的问题：** 版本升级后 `init` 检测到 hook 已存在会跳过，无法更新旧版 hook。

## 改进

- 版本号更新至 1.1.1
- pre-push hook 模板更新：部署改为后台执行，输出重定向到日志文件
- `cmd_init()` 安装 hook 后自动替换 `__LOCAL_DEPLOY_HOME__` 占位符为实际路径
- `usage()` 更新：init 命令说明为 `init [--force]`，新增 `log` 命令说明
