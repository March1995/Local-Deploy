# 功能迭代

## v1.1.0（已完成）

### 1. 支持单独执行 Maven 构建流程和 Docker 构建流程 ✅

- 新增 `--skip-docker` 参数：仅执行 Maven 构建
- 新增 `--skip-maven` 参数：仅执行 Docker 构建和部署
- 在进行其他环境打包时，无法连接 Docker，所以只需要执行 Maven 构建流程
- 部分情况下，Maven build 成功后，由于 Docker 的配置错误，只需要再执行 Docker 构建流程就好了，无须重复打包

### 2. 支持对 Docker 容器的操作 ✅

- 新增 `container` 命令，支持子命令：`stop`、`start`、`restart`、`rm`、`logs`、`status`
- 容器名称自动从配置推导（`{artifactId}-{profile}`）
- 自动设置 `DOCKER_HOST` 连接远程 Docker

### 3. 添加参数控制是否开启自动构建 ✅

- 新增 `auto_deploy` 配置项，默认 `true`
- 设为 `false` 时，`git push` 触发的 Hook 会跳过部署
- 手动 `local-deploy deploy` 不受影响
