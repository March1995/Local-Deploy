# 功能迭代

## v1.1.1（已完成）

### 1. 使用第三方工具 git push 后，后台自动执行日志查看功能 ✅

- pre-push hook 使用 `nohup ... &` 后台执行部署，不阻塞 git push
- 日志放在 init 生成的 yml 文件同级目录（项目根目录 `local-deploy.log`）
- 使用覆盖策略，每次执行时，都覆盖之前的日志文件
- 新增 `log` 命令，支持查看最近一次部署日志
- 支持 `--follow`（`-f`）参数，实时跟踪日志输出

### 2. Hook 自动写入 Local-Deploy 安装路径 ✅

- pre-push hook 模板新增 `LOCAL_DEPLOY_HOME` 变量（占位符 `__LOCAL_DEPLOY_HOME__`）
- `cmd_init()` 安装 hook 时自动用 `sed` 将占位符替换为实际的 `SCRIPT_DIR`
- hook 查找 `local-deploy.sh` 的优先级：全局命令 → 项目根目录 → `LOCAL_DEPLOY_HOME`
- 解决未全局安装时 hook 找不到 `local-deploy.sh` 的问题

### 3. `init` 命令支持 `--force` 强制覆盖 ✅

- 新增 `--force`（`-f`）参数，强制覆盖已有的配置文件和 pre-push hook
- 解决版本升级后 `init` 无法更新旧 hook 的问题
